CSDN上大量的相关专栏应该比本项目更好，附带的OJ也比本项目简单的CTest更精确。

本项目主要只是本人对题目的重新归类和小量补充。

分类规则：

    不同目录必然有交叉

    模拟题也可以归类为查找，排序，核心思想是找到合适的数据结构，所以也可以归类到对应的数据结构中，特指数据结构和算法都不突出的情况

    如果模拟题的规律性比较明显，且不需要动态表辅助，就纳入贪心（最大子序和问题也是这两个概念的交叉点）

    个人按难度排列顺序为：

    二进制

    模拟

    线性表

    —链表

    —双指针

    ——二分

    ——滑动窗

    —字符串

    ——一般操作

    ——正则

    ——KMP

    —优先队列

    —单调栈

    二叉树和递归

    BFS迭代法

    回溯

    动态规划

    图论
    
    —最小生成树

    —最短路

    贪心

    如果一道题目有多个分类 将题目归类到排序最靠后的那一类分类中
